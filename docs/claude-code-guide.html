Hướng dẫn dùng Claude Code cho dự án

Dự án đã được cấu hình sẵn .claude/ + plugin Medusa chính thức. Tài liệu này hướng dẫn cả team cài đặt và khai thác toàn bộ công cụ — từ quyền, hook tự format, lệnh tắt, đến plugin dựng tính năng Medusa.

⚠️
Bẫy quan trọng — 2 thư mục lồng nhau. Gốc git là thư mục ngoài; workspace pnpm là thư mục hf-medusa-store/ bên trong. Mọi lệnh pnpm/turbo phải chạy ở thư mục trong. (Claude đã biết điều này qua CLAUDE.md.)
1

Cài đặt cho thành viên mới

Cấu hình dự án (CLAUDE.md, quyền, hook, lệnh tắt, rules) đã nằm trong repo — kéo code về là có. Chỉ cần cài plugin Medusa một lần cho máy bạn.

Terminal — trong thư mục dự án

Mở Claude Code ở terminal thật rồi gõ lần lượt (mỗi lệnh Enter riêng):

/plugin marketplace add medusajs/medusa-agent-skills
/plugin install medusa-dev@medusa

Khi hỏi phạm vi, chọn “Install for all collaborators (project scope)”. Sau đó chạy /reload-plugins (hoặc mở lại Claude) để áp dụng.

💡
Cấu hình enabledPlugins đã được commit sẵn, nên nhiều máy sẽ tự nhận plugin. Nếu chưa, chạy 2 lệnh trên là xong. MedusaDocs (tra cứu tài liệu) đi kèm plugin — không cần cài riêng.
2

Dùng hằng ngày — Claude tự lo

Bạn không phải gõ lệnh đặc biệt. Mỗi phiên làm việc, Claude tự đọc CLAUDE.md và .claude/rules/project-conventions.md nên đã biết:

  • Dùng pnpm (không npm/yarn) và chạy lệnh ở đúng thư mục workspace bên trong.
  • suggestive-selling là khuôn mẫu module; tham chiếu module khác qua Link Module, không dùng FK.
  • Storefront chỉ gọi SDK trong lib/data/*; cổng 8008; Redis là tùy chọn; react-router ghim 6.30.4.
  • Comment nên trích mục SRS (vd SUGG-001); quy ước đặt tên test.

Quyền — bớt hỏi xin phép

Các lệnh an toàn chạy không cần bấm “cho phép”; lệnh nguy hiểm luôn bị chặn.

.claude/settings.json

Cho phép (không hỏi)

git statusgit diffgit loggit showgit branchlscatfindgreppnpm lintpnpm testpnpm build

Luôn chặn

rm -rfgit push

Muốn thêm/bớt: gõ /permissions. Quyền riêng máy bạn (không commit) để trong .claude/settings.local.json.

4

Hook tự động format

Sau khi Claude sửa/tạo file .ts .tsx .js .json .css .md… → Prettier tự chạy làm đẹp code. An toàn: chỉ đụng file trong workspace, không bao giờ chặn thao tác. Bạn không cần làm gì.

6

Plugin medusa-dev — sức mạnh chính

Cứ mô tả tính năng bằng lời, Claude tự vận dụng kiến thức Medusa chuẩn (module, API, workflow, admin UI, storefront) để làm.

Lệnh pluginTác dụng
/medusa-dev:db-generate <module>Tạo migration cho một module
/medusa-dev:db-migrateChạy migration
/medusa-dev:new-user <email> <pass>Tạo tài khoản admin

Tra cứu tài liệu: MCP MedusaDocs cho Claude tra tài liệu Medusa chính thức khi code — tự động, bạn không phải làm gì.

Ví dụ prompt“Tạo module wishlist: khách lưu sản phẩm yêu thích, có API thêm/xóa/lấy danh sách.”
Ví dụ prompt“Thêm widget vào trang chi tiết sản phẩm trong admin để quản lý sản phẩm liên quan.”
Ví dụ prompt“Tích hợp API reviews vào storefront, hiển thị review ở trang sản phẩm kèm phân trang.”
